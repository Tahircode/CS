## class

 - it is a blueprint that is used to create objects.
 - It's a template or a definition for creating objects. It defines what attributes and methods every object of that type will have.
 - class Car is the blueprint. It specifies that every car will have a color, a speed, and an accelerate() method.
- A class itself doesn't exist in memory. It's just the design.

- a class can contain variables, function etc
## object
- it is a perticular instance of a class

- object is an entity that has state(color,length,..called data members) and behaviour(can write(), dance(), run()... called member functions/methods).


- An object is the actual thing created from the class blueprint. It's a concrete entity that lives in memory and holds actual values.
- myRedTesla = new Car() creates an object. myRedTesla is an instance of the Car class. Its color might be "Red" and its speed might be 0.
- yourBlueFord = new Car() is another, distinct object. Its color is "Blue".
