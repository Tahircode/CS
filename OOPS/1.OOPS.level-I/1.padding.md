**1. What is padding?**


Padding is the extra unused memory added by the compiler between data members or at the end of a structure/class to satisfy alignment requirements of the CPU.

- Why padding is needed?

Modern CPUs access memory faster when data is stored at addresses that are multiples of their size (alignment).

Example:

int → 4-byte aligned

double → 8-byte aligned

If alignment is violated:

Performance degrades

On some architectures → crash (bus error)

Example
```
struct A {
    char c;     // 1 byte
    int i;      // 4 bytes
};
```

Memory layout:

| c | pad pad pad | i i i i |


char c → 1 byte

3 bytes padding added

int i starts at 4-byte boundary

📌 sizeof(A) = 8 bytes, not 5

Padding at end of structure
```
struct B {
    char c;
    double d;
};
```
| c | pad x7 | d (8 bytes) |


📌 Total size = 16 bytes

How to reduce padding?

- Reorder members from largest → smallest:
```
struct Optimized {
    double d;
    int i;
    char c;
};
```
- Interview one-liner

Padding is unused memory added by the compiler to align data members for efficient CPU access.

**2. What is greedy alignment?**

Greedy alignment is the strategy used by the compiler where:

Each data member is placed at the next available address that satisfies its alignment, without considering future members.

The compiler is “greedy” — it aligns each member independently.

Example
```
struct C {
    char c;     // 1 byte
    double d;   // 8 bytes
    int i;      // 4 bytes
};
```

Step-by-step layout (Greedy):

char c → offset 0

double d → must start at multiple of 8

→ compiler inserts 7 bytes padding

int i → placed at next multiple of 4

| c | pad x7 | d (8 bytes) | i (4) | pad x4 |


📌 Total size = 24 bytes

Even though a better layout exists, compiler does not rearrange members.

Why compiler uses greedy alignment?

Predictable layout

Faster compilation

ABI compatibility

Programmer controls ordering

- Interview one-liner

Greedy alignment is the compiler’s strategy of aligning each member independently at the first valid aligned address, which often introduces padding.
