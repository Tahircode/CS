1. What is class?
- A class is the fundamental blueprint in Object-Oriented Programming. It is a user-defined data type that encapsulates data (attributes) and the functions that operate on that data (methods or behaviors).

2. What are the member variables or class level variables?
-  Member Variables (or Instance Variables)
These are the variables declared inside a class without the static keyword. Each object (instance) of the class gets its own separate copy of these variables. Changing a member variable in one object does not affect it in another object.

 - Class Level Variables (or Static Variables)
These are variables declared inside a class with the static keyword. There is only one copy of a static variable, which is shared by all objects of that class. It belongs to the class itself, not to any single instance.
Purpose: To represent data that is common to all objects of a class, or to keep a global count or shared state.

   - string color; //member variable
   - static int carCount; // static variable / class level varaiable

3. What is the difference between class and struct in c++?
 - Feature                        class                                           struct
Default Access Specifier	     private	                                      public
Default Inheritance Type	private inheritance	                            public inheritance

- Semantic Meaning / Convention: 
	class: Used for complex types that have private data and methods to control access (encapsulation). They represent objects with behavior and invariants to protect.	                              
    struct: Used for simple data aggregates, or "Plain Old Data" (POD) structures. They are primarily containers for data with all members publicly accessible. 


- The Core Difference: The only technical difference is the default access. A class with all members public is functionally identical to a struct.

4. What are the access modifiers?

- Access modifiers are keywords in a class that define the visibility and accessibility of its members. They are the primary tool for implementing Encapsulation.

5. When should we use class instead of struct and vice-versa?

- Use a struct when:

. You are creating a simple data container.
. All or most of its members are intended to be publicly accessible.
. It has little to no functionality (methods), other than perhaps simple constructors for convenience.
. It does not need to protect any invariants.
. Examples: Vector2D, RGBColor, ConfigurationSettings.

- Use a class when:

- You need to bundle data with functions that operate on that data.
- You have private member variables that need to be protected from direct external access.
- You need to enforce business rules or invariants (e.g., balance must be non-negative).
- You are designing a component with clear responsibilities and a public API.
You expect the class to be extended through inheritance.
- Examples: DatabaseConnection, UserAuthentication, FileParser, Car.

- A simple rule of thumb: If you find yourself writing the private: keyword, you should be using a class
