# Copy Constructor.
- A copy constructor is a special constructor that is used to create a new object as a copy of an existing object of the same class. It's essentially the official "cloning" mechanism for a class.

- The compiler automatically generates a copy constructor for you if you don't write one yourself.
- This compiler-generated version is called the implicit copy constructor.

**The Canonical Signature:**

cpp:
ClassName(const ClassName& other);

. It takes a single argument: a reference to an object of the same class.
- It's a const reference because the constructor should not (and must not) modify the object it's copying from.
- It must be a reference. If it were passed by value (ClassName other), the compiler would have to make a copy of the argument to pass it to the constructor, which would require... calling the copy constructor! This would lead to infinite recursion.

# When is the Copy Constructor Called?
It's called in three distinct situations:

1. When you explicitly initialize a new object from an existing one:

MyClass obj1;
MyClass obj2 = obj1; // Calls copy constructor
MyClass obj3(obj1);  // Also calls copy constructor

2. When you pass an object to a function by value:

void myFunction(MyClass obj) { /* ... */ }
MyClass mainObj;
myFunction(mainObj); // A copy of mainObj is made to pass to the function.

3. When you return an object from a function by value:

MyClass myFunction() {
    MyClass localObj;
    return localObj; // A copy of localObj is made to return.
}

## error :-=> infinite loop

1. 
public Hero(Hero temp) {
    this(temp);   // ❌ calls same constructor again
}

What happens?
Hero(Hero temp)
  → calls Hero(Hero temp)
      → calls Hero(Hero temp)
          → ...

🔥 Infinite recursion → StackOverflowError

2. Creating a new object inside copy constructor
public Hero(Hero temp) {
    Hero h = new Hero(temp); // ❌
    this.a = h.a;
    this.b = h.b;
}

What happens?
new Hero(temp)
  → inside constructor → new Hero(temp)
      → inside constructor → new Hero(temp)
          → ...


🔥 Infinite object creation

3.  Chaining via this(...) incorrectly

public Hero(Hero temp) {
    this(new Hero(temp)); // ❌ indirect recursion
}

Same problem → constructor keeps calling itself.
