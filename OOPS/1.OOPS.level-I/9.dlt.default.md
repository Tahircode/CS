## Why Delete the Default Constructor? (A Deeper Look)

- the default constructor is the one that takes no arguments. Deleting it (MyClass() = delete;) is a powerful design choice to enforce rules and prevent invalid objects.

- The core reason is to force the user of your class to provide necessary information for the object to be valid.

class DatabaseConnection {
private:
    std::string connectionString;
    bool isConnected;

public:
    // We delete the default constructor.
    DatabaseConnection() = delete; 

    // We ONLY provide a constructor that requires the connection string.
    DatabaseConnection(std::string connStr) {
        connectionString = connStr;
        // Logic to actually connect to the database...
        isConnected = true;
        std::cout << "Connected to database." << std::endl;
    }
};
