## OOPS
Object-Oriented Programming or OOPs refers to languages that use objects in programming. it aims to implement real-world entities like inheritance, hiding, polymorphism, etc in programming. The main aim of OOP is to bind together the data and the functions that operate on them so that no other part of the code can access this data except that function.

A program is a collection of objects that interact with each other. Each object is a self-contained unit that bundles its own data (attributes) with the behavior (methods) that operates on that data.

OOPs is a technique or pattern that revolves around object or entity

#### what is 'S' in OOPS

In OOPS (Object-Oriented Programming), the “S” usually refers to Single Responsibility Principle from the SOLID principles.

SOLID is a set of design principles used to write clean and maintainable code.

S — Single Responsibility Principle (SRP)

Definition:
A class should have only one responsibility or one reason to change.

This means a class should do only one job.

❌ Bad Example (Violates SRP)

```
class Report {
public:
    void generateReport() {
        // generate report
    }

    void saveToFile() {
        // save report to file
    }
};
```

Here the class does two things:

Generates report

Saves report

So it has two responsibilities.

✅ Good Example (Follows SRP)
```
class ReportGenerator {
public:
    void generateReport() {
        // generate report
    }
};

class ReportSaver {
public:
    void saveToFile() {
        // save report to file
    }
};
```

Now:

ReportGenerator → only generates report

ReportSaver → only saves report

Each class has one responsibility.
