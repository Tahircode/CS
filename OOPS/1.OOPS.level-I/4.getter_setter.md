## getter and setter
- Getters and setters are public methods used to read (get) and modify (set) private data members of a class.

- They are a core part of Encapsulation in OOP.

- Why do we need getters and setters?

. Encapsulation

   Protect internal data from direct access

. Validation

   Control what values can be set

. Abstraction

   Internal representation can change without affecting users

. Security

   Prevent illegal or unsafe state

. Maintainability

   Central point of modification

📌 Interview keyword: “controlled access to data”

**JAVA**
class Account {
    private int balance;

    // Getter
    public int getBalance() {
        return balance;
    }

    // Setter
    public void setBalance(int balance) {
        if (balance >= 0) {
            this.balance = balance;
        }
    }
}
//usage:
Account acc = new Account();
acc.setBalance(1000);
System.out.println(acc.getBalance());


**C++**

class Account {
private:
    int balance;

public:
    int getBalance() const {
        return balance;
    }

    void setBalance(int b) {
        if (b >= 0)
            balance = b;
    }
};


# 📌 const ensures getter does not modify the object.

Real-world analogy (interview-friendly)

- A getter is like reading your bank balance.
-  A setter is like depositing money through bank rules, not directly editing the database.


## another way to declare getter/setter

1️⃣ Lombok Annotations

- Lombok is a Java library that auto-generates boilerplate code (getters, setters, constructors, etc.) at compile time using annotations.

1. @Getter and @Setter

 import lombok.Getter;
 import lombok.Setter;

 @Getter
 @Setter
 class User {
    private String name;
    private int age;

✔️ Generates:

getName(), setName()
getAge(), setAge()

2. Field-level usage
class User {
    @Getter
    private String name;

    @Setter
    private int age;
}
 
✔️ Fine-grained control

3. @Data (MOST USED)
import lombok.Data;

@Data
class User {
    private String name;
    private int age;
}


📌 Generates:

Getters

Setters

toString()

equals()

hashCode()

Required constructor

⚠️ Interview tip:

@Data should be used carefully in domain models.
