# propeerties of Copy Constructor

- same name as class name and no return type;
- it takes refernce to an object as argument
      Strategy(Strategy & obj)
- it is invoked automatically when a new object is created as a copy of existing object
  Strategy obj1;
  Strategy obj2(obj1);

- user can provide its own copy contructor, if not provided class has default implementation

- default copy contructor creates Shallow copy
- to create deep copy ,user needs to provide its own implementation
- copy constructor can be overloaded
- when passing object as an argument to any function calls the copy constructor
 -> pass by value: creates copy
 -> pass by reference: modification to the original object, no copy
 -> pass by cosnt reference: no copy is made. The const keyword is a contract that promises the function will not modify the object.
-> pass by pointer: You pass the memory address of the object. The function receives a copy of this address. You must use the -> (arrow) operator or the * (dereference) operator to access the object's members.
