## 4 pillers of OOPS
- pillars of OOP are Encapsulation, Abstraction, Inheritance, and Polymorphism. They help in building secure, maintainable, reusable, and flexible software. Ignoring them leads to tightly coupled, hard-to-maintain, and error-prone code.

| Pillar        | Purpose         | If Ignored             |
| ------------- | --------------- | ---------------------- |
| Encapsulation | Protect data    | Unsafe, unmaintainable |
| Abstraction   | Hide complexity | Rigid, messy code      |
| Inheritance   | Reuse code      | Duplication            |
| Polymorphism  | Flexibility     | Condition-heavy logic  |



**🔷 C++: Can a class itself be public / private / protected?**

- ✅ At global (namespace) scope

NO access specifiers allowed

public class A { };    // ❌ INVALID
private class B { };   // ❌ INVALID
protected class C { }; // ❌ INVALID

✔ A global class is always public by default.

- ✅ Inside another class (nested class)

✔ YES — nested classes can be public, protected, or private.

class Outer {
public:
    class PublicInner { };

protected:
    class ProtectedInner { };

private:
    class PrivateInner { };
};


📌 Accessibility:

PublicInner → accessible everywhere

ProtectedInner → accessible in derived classes

PrivateInner → accessible only inside Outer

🔑 C++ one-line summary

In C++, only nested classes can be public, protected, or private.
Top-level classes cannot have access specifiers.

**🔶 Java: Can a class itself be public / private / protected?**

- ✅ Top-level class

✔ Only public or default (package-private) allowed

public class A { }   // ✔ allowed
class B { }          // ✔ default (package-private)

private class C { }  // ❌ NOT allowed
protected class D { } // ❌ NOT allowed

📌 Rule:

Only one public class per file

File name must match public class name

- ✅ Nested (inner) classes

✔ YES — all access levels allowed

class Outer {
    public class PublicInner { }
    protected class ProtectedInner { }
    private class PrivateInner { }
}

🔑 Java one-line summary

In Java, top-level classes can be public or package-private only.
Inner classes can be public, protected, or private.
