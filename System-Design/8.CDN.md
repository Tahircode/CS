## CDN ( content delivery network)
- A CDN is a network of geographically distributed servers that deliver content to users from the nearest location, reducing latency and load on the origin server.

          ___ Static Contents ( Images , Videos, CSS, JS etc)
         |
Website - 
         |___ Dynamic Contents ( Changes according to uer input)

#### how CDN works:
- user visit a website
- CDN server closest to user -> static content
- if not in CDN -> original server

#### Things to consider for using CDN
- cost: get rid of infrequently used contents inside CDN ( bcz they charge)
- expiry-time: not too low( we dont want load on web-server) , not too high ( stale data)
- CDN fallback/back-up: backup when CDN fails ( we must should have mechanism to fetch data from webserver during CDN failure)
- invalidating files: 
    - API ( CDN venders: provided by CDN to invalidate files)
    - versioning ( example: image.png?v=2 , v=2 -> CDN will know first got invalidated , requests for second one)
