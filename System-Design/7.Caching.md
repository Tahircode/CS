## Cache:
- Temporary storage
- Stores
  - results of expensive results
  - frequently accessed data
- we can serve requests more quickly

![alt text](<0.public/image copy 9.png>)

#### interaction with Cache:
- Cache servers provide API's
Example:
- cache.set('my-key','Hello',3600seconds);
- cache.get('my-key');

#### Things to consider for using Cache:
- Data read frequently + modifying infrequently
- Expiration Policy:
     - Time: 
          - not too low: Data reload from DB
          - not too high: Stale data ( Stale data is outdated data)
- Consistency : b/w Cache & DB
- Multiple cache servers across different data centers to avoid SPOF ( Single Point of Failure (SPOF) is any component in a system that, if it fails, causes the entire system to stop working. If that one server crashes,
→ whole system goes down: That server is a SPOF.)
- Eviction Policy: when data gets full
    - LRU : least recently used
    - LFU : least frequently used
    - FIFO: first in first out
