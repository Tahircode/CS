## Data-Centers & geoDNS
as the service grows internationally : lot of users
- to improve availabilty & better experience across wider GEOGRAPHICAL areas. we should support MULTIPLE DATA CENTERS 

#### geoDNS
- route to closest data-center
eample:
- US(east) - [data-center-1]
- US(west)- [data-center-2]
split-traffic by routing: with the help of geoDNS
- x% -> US(east)
- (100-x)% -> US(west)

so what geoDNS does:
- takes domain-name
- return the ip-address (closest) based on users location
