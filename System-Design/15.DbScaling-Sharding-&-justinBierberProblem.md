#### shard vs slave
A shard is a database that stores only a subset of the total data.
-  Instead of:
   - All users → One DB
- We split:
    - Users 1–1M → Shard A
    - Users 1M–2M → Shard B

A slave (replica) is a database that stores a full copy of another database (primary).

#### Micor-services:
Microservices is an architecture style where an application is built as a collection of small, independent services, each responsible for a specific business function.

Instead of:
- One big application (Monolith)

You build:
- User Service
- Order Service
- Payment Service
- Notification Service

Each runs independently.

#### how micorservice relates to message queue
Microservices need to communicate.

Two ways:
- 1. Synchronous Communication
    - Service A → HTTP → Service B → Response
        - Blocking
        - Tight coupling
        - If B is down → A fails
- 2. Asynchronous Communication (Message Queue)
    - Service A → Queue → Service B
       - A does not wait
       - Loose coupling
       - More reliable
       - Handles spikes better
