## Load Balancer:

as you can see ![alt text](<0.public/image copy 3.png>) , users are directly connected to the server, so when server goes down/offline -> impact users, and Also when many users access at the same time:
 - web server load limit
 - slow response
 - fail to connect

 so the best technique to handle this problem is the load balancer
 
  ![alt text](<0.public/image copy 5.png>)

 What it does:
 - evenly distributes the traffic
 - Users connect to Load-Balancer via public-IP
 - and load balancers communicated with web-servers via private-IP

 Now: if server 1 goes offline, server 2 will get all the traffic

Common load balancing strategies: 
- Round Robin – each server gets turns
- Least Connections – server with least load
- IP Hash – same client → same server
- Weighted – stronger servers get more traffic

Types of load balancers:
- Software (Nginx, HAProxy)
- Hardware (dedicated appliances)
- Cloud-managed (AWS ALB/ELB)
