## What is System-Design:

- System design is the process of defining the architecture, components, data flow, and interactions of a software system so that it meets functional requirements (what it should do) and non-functional requirements (scale, reliability, performance, security, cost).

#### Single server:

![alt text](<0.public/image copy.png>)

![alt text](<0.public/image copy 2.png>)

A single server setup is an architecture where one physical or virtual machine handles everything:
- Client requests
- Application logic
- Database
- File storage
    
    - All components run on one server.

What runs on the single server
- Web server (HTTP)
- Backend application
- Database
- Storage
- Background jobs (if any)

Limitations
- Single point of failure (if server dies, everything dies)
- Cannot scale well
- Limited CPU, RAM, disk
- Difficult to handle high traffic
- Risky updates (downtime)

- so a single server aint enough to handle larger number of requests/users
