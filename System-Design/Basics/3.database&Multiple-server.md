## which Database to use:

Types:
1. Relational db:
2. Non-relational db

#### Rational DB
- RDBMS or SQL Database
- example: oracle, postGreSQL, MYSQL
- represent and store data in Tables and Rows
- can perform JOIN operations using SQL on different tables

#### Non-relational db
- No-SQL Database
- example: cassendra,  amazon DynamoDB etc
- grouped into 4 categories
    - Graph Stores,
    - Key-Value Stores,
    - Column Stores,
    - Document Stores
- JOIN operation not supported

#### which database should we use:
- for most developers -> rational db
- but there are many cases in which R-DB not suitable

#### when to think of Non-Rational Database:
- your application super low latency
- when the data is unstructured
- you only need to serialze and de-serialize data( JSON, XML, YAML etc) , or you dont need schema.
- when you need to store MASSIVE amount of data
- when you need to define data type etc , no restrictions
- for storing a large number of data about users/customers
