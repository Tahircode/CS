## what is scaling:
- Scaling is the ability to increase a system’s capacity so it can handle more users, more requests, or more data without breaking.


#### Types:

![alt text](<0.public/image copy 4.png>)

1. Vertical Scaling:
- Scale-up
   - adding more power to your server
   - example: add more CPU, RAM

Limitations:
- impossible to add unlimited CPU and RAM to a single server
- No backup (failover)
    - of one server goes down, the whole service/website goes down completely

2. Horizontal Scaling:
- Scale-out
   - Add more machines
   - Distribute load
   - example: 1 server -> 10 servers

 - Requires coordination (load balancing).




 #### which one to use:
 
 Vertical Scaling
 - when trafic is low
 - simplicity is its advantage

 Horizotal Scaling
 - For large scale application
 - Huge Trafic
