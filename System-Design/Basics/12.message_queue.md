 - Develop
 - Deploy
 - Maintain

 how they communicate with each other
 - through MESSAGE QUEUE
 message queue provide communication and coordination for them

#### message queue
- supports asynchronous communication

- producer can produce a messge to queue , even when the consumer is unavailable
- consumer can read message from queue , even when the producer is unavailable

Example: Photo customization service

![alt text](<0.public/image copy 20.png>)

Producer vs Consumer(servers)
- independent 
- scaled independently
     - queue size large -> add more consumers/workers
     - if queue is empty mostly: reduce consumers/workers

![alt text](<0.public/image copy 21.png>)
- dead letter queue: deal with the messages that fail too many times:
    - so, instead of infinitly retrying
    - Move message to Dead Letter Queue
        - Prevent blocking normal queue


Key properties
- FIFO (usually)
- Durable storage
- Retry mechanism
- Acknowledgment system

Common use cases
- Email sending
- Order processing
- Notifications
- Logging
- Microservices communication

if Queue is crashed , we wont lost our data : because the data aint stored in RAM rather it is stored in disc which is persistant


#### Types of queues:
- FIFO base
- Priority-queue
#### queus can also be push based or pull based
pull: they wait for workers/consumers to pull message

push: they actively send messages to comsumers to be processed

#### examples of message queues:
- RobbitMQ
