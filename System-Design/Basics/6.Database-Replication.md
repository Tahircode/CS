## Database replication

![alt text](<0.public/image copy 6.png>)

- Database replication is the process of copying data from one database (primary) to one or more other databases (replicas) so the same data exists in multiple places.

                 ___ Master - ( Original DB )
                |
DB Replication -
                |
                 --- Slave - Copy( Relica )

![alt text](<0.public/image copy 7.png>)

#### Master DB
- only write operations
- insert , delete, update ( all data-modifying cmds)
- changes inside master DB get also updated inside Slave DB

#### Slave DB
- Read opearions
- gets copies of data from master

Most Applictions :=> read/write Ratio is HIGH
- usually: no. of Slave > no. of Master

#### Advantages of DB Replication:
- Better performance: allows more queries to be processed in parallel
- Relaibility: Data preserved if 1 DB crashes
- High availability of service

#### question: what if slaves shutdown
we will process the read operartions from master DB, mean while we will add a new slave DB 

#### what if master DB goes down?
- i will promote a slave DB into Master DB
- mean while we will add new slave db to it.

In real life promoting a Master DB is complicated
- there can ba a case that slave DB cant do write operations
