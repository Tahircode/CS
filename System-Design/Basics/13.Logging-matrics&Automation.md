## Logging, Matrix , Automation
- small websites -> few servers -> not neccessity
- busy & popular websites -> essential

#### Logging
Logging is recording events that happen inside a system.

![alt text](<0.public/image copy 22.png>)

- monitoring error logs
- per server level
- centralized logging
    - In distributed systems:
        - App Servers → Log Collector → Log Storage
        - Logs from multiple machines are aggregated in one place.
    - example: new Relic
- It answers:
   - What happened?
   - When did it happen?
   - Why did it fail?
- What gets logged
   - Errors
   - Warnings
   - User actions
   - API calls
   - System events

Log levels
- DEBUG
- INFO
- WARN
- ERROR
- FATAL

#### Metrics
Metrics are numerical measurements that track system health over time.
- Business gain insights
- aware of health of our service
- Host level matrics: CPU, memory etc
     - example: Grafana
- Aggregate level matrics:
    - performance of DB, Cache etc
- Key Business Matrics:
    - daily active users, revenue etc

Examples:
- CPU usage
- Memory usage
- Requests per second
- Error rate
- Latency (p95, p99)

#### Automation:

Automation means letting software perform repetitive operational tasks instead of humans.
- when system gets big and complex
- autommation tools
- improve productivity
- Contineous Integration
     - code checkIn verified thrugh automation
- automating buils, tests, deplyments etc
    - example: jenkins jobs

Where automation is used
- CI/CD deployments
- Scaling servers automatically
- Auto-restart failed services
- Infrastructure provisioning
- Backup scheduling

Example
- If CPU > 80%:
    - → Automatically add new server

No human needed.

- message queue is also an automated tool
