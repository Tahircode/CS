## INDEXING
Indexing is a technique used to speed up data retrieval operations in a database by creating a separate data structure that stores references to table records.
it minimizes the number of disk accesses

Index -> it is a data-structure , to locate the record on DB

#### Note : 
- there are basically 2 types of tables : one is index table and another is main table 

🧠 Critical Example

Suppose we have a table:

STUDENT

| RollNo | Name  | Dept |
| ------ | ----- | ---- |
| 101    | Tahir | CS   |
| 102    | Ali   | EE   |
| 103    | Sara  | ME   |

If you run:

SELECT * FROM Student WHERE RollNo = 103;

Without index:

DBMS scans every row → Full Table Scan.

With index:

DBMS directly jumps to the location of RollNo = 103.

Much faster.

#### components of index file

              search key & reference 
                 /             \
          (attribute)      (pointer, pointing to a perticular block)

## Types of Indexing

#### 1. Primary Index
- An index created on the primary key of a table.
- it is defined on the ordered data file
Characteristics:
- Unique values
- Data is sorted based on primary key

| RollNo | Name  |
| ------ | ----- |
| 101    | Tahir |
| 102    | Ali   |

Primary index helps quickly locate records.

 Types of Primary Index

 a. Dense index:
 - A Dense Index contains an index entry for every record in the table.
 - Every search key value has a corresponding index entry.

 b. Sparse index
- A Sparse Index contains index entries only for some records, not all.
- Usually one entry per data block or group of records.

🧠 Critical Example

Suppose we have a table:

| ID | Name |
| -- | ---- |
| 10 | A    |
| 20 | B    |
| 30 | C    |

Dense Index

Index table:

| Key | Pointer  |
| --- | -------- |
| 10  | → Record |
| 20  | → Record |
| 30  | → Record |

Every record has an index entry.

Sparse Index

Index table:

| Key | Pointer  |
| --- | -------- |
| 10  | → Block1 |
| 30  | → Block2 |
| 50  | → Block3 |

Each entry points to a block of records.

DBMS then searches inside that block.

🎯 Use Case

Dense Index Used when:
- Fast lookups required
- Secondary indexes
- Data not sorted

Sparse Index Used when:
- Data is sorted
- Primary index
- Storage optimization needed

#### 2. Secondary Index

An index created on non-primary key attributes.

This is two level indexing

Example:
- Index on Name column.
- Multiple students may have same name.

#### 3. Clustered Index ⭐

The table data is physically stored in the same order as the index.

Characteristics:
- Only one clustered index per table
- Data rows are sorted

Example:

Students stored ordered by RollNo.

#### 4. Non-Clustered Index

Index is stored separately from the table and contains pointers to data rows.

Characteristics:
- Multiple non-clustered indexes allowed
- Table data not physically ordered
