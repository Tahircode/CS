## Relational Model
Organises the data in the form of Tables

consists of: multiple tables
- have unique name
- Rows (Tuples)
- Columns (Attributes)
- Keys
- Relationships using foreign keys

RDBMS (Relational Database Management System) is the software that implements the relational model.
 
Relational Model = Theory

RDBMS = Practical Implementation of that theory

#### degree:
no. of colomuns in a row

#### cardinality
no. of rows

### KEYS
Uniquely identify a tuple (row) in a relation (table)

Establish relationships between tables

#### Super key
A Super Key is any set of attributes that uniquely identifies a tuple( single row).

can be:
- {RollNo}
- {Email}
- {RollNo, Name}
- {RollNo, Email}
- NULL

#### Candidate key
A Candidate Key is a minimal super key.
- example: {RollNo} or {Email} or NULL(dont prefer)

#### Primary key
A Primary Key is the candidate key chosen to uniquely identify tuples.
- Cannot be NULL
- Only one per table

#### Alternate key
Candidate keys that are not selected as primary key.

example: If RollNo is PK → Email becomes Alternate Key.

#### Foreign Key
A Foreign Key is an attribute that references the Primary Key of another table.
    
#### Composite key
A key formed using two or more attributes together.

#### compound key

## Integrity Constraints

Integrity constraints prevent invalid data from entering the database.

### 1. Domain Constraint

Ensures that attribute values must belong to a valid domain (set of allowed values).

example: marks must be integer and between 0-100
```
Marks INT CHECK (Marks BETWEEN 0 AND 100)
```

### 2. Key Constraint
Ensures that all values of a key are unique.

No two students can have the same RollNo.
```
RollNo INT PRIMARY KEY
```

### 3. Entity Integrity Constraint ⭐⭐⭐

Primary Key:
- Must be unique
- Cannot be NULL

### 4. Referential Integrity Constraint ⭐⭐⭐
Foreign Key must:
- Match a Primary Key in referenced table
- Or be NULL (if allowed)

### 5. NOT NULL Constraint
Prevents NULL values in a column.
```
Name VARCHAR(50) NOT NULL
```

### 6. UNIQUE Constraint
Ensures no duplicate values.
```
Email VARCHAR(100) UNIQUE
```

### 7. CHECK Constraint

Enforces custom condition.
```
CHECK (Age >= 18)
```
### 8. DEFAULT Constraint
Provides default value.
```
Status VARCHAR(10) DEFAULT 'Active'
```
