## SQL

SQL (Structured Query Language) is a standard language used to communicate with relational databases to:

Store data

Retrieve data

Modify data

Manage database structure

Control access

👉 In simple words:

SQL is the language used to interact with an RDBMS.

Examples of systems using SQL include MySQL, PostgreSQL, and Oracle Database.

## 🧩 Types of SQL Commands

SQL commands are grouped into categories.

1️⃣ DDL (Data Definition Language)

Used to define database structure.

Examples:

CREATE TABLE Student (
   ID INT PRIMARY KEY,
   Name VARCHAR(50)
);

Common commands:

CREATE

ALTER

DROP

TRUNCATE

2️⃣ DML (Data Manipulation Language)

Used to modify data inside tables.

Examples:

INSERT INTO Student VALUES (1, 'Tahir');
UPDATE Student SET Name = 'Ali' WHERE ID = 1;
DELETE FROM Student WHERE ID = 1;

Commands:

INSERT

UPDATE

DELETE

3️⃣ DQL (Data Query Language)

Used to retrieve data.

Example:

SELECT * FROM Student;

Main command:

SELECT

4️⃣ DCL (Data Control Language)

Used to control database access.

Example:

GRANT SELECT ON Student TO user1;
REVOKE SELECT ON Student FROM user1;

Commands:

GRANT

REVOKE

5️⃣ TCL (Transaction Control Language)

Used to manage transactions.

Example:

COMMIT;
ROLLBACK;
SAVEPOINT sp1;

Commands:

COMMIT

ROLLBACK

SAVEPOINT

#### alter

ALTER is a SQL command used to modify the structure of an existing table in MySQL.

It allows you to add, delete, or change columns without deleting the table.

for e.g:

adding constraints like unique, not null, primary key, check, foreign key

```sql
alter table student add unique(s_name);
```

#### check

CHECK is a constraint used to restrict the values that can be inserted into a column.

It ensures that the data satisfies a specific condition.

```sql
CREATE TABLE student(
  s_id INT,
  s_name VARCHAR(50),
  age INT,
  CHECK (age >= 18)
);
```
#### GROUP BY

used to group rows that have the same values in a column and then apply aggregate functions to each group.

Aggregate functions include:
- COUNT()
- SUM()
- AVG()
- MAX()
- MIN()

```sql
select s_branch, count(s_name) from student group by s_branch;
```

## set theory in SQL

Critical Example: Suppose we have two tables.

STUDENT_CS

| ID | Name  |
| -- | ----- |
| 1  | Tahir |
| 2  | Ali   |

STUDENT_EE

| ID | Name |
| -- | ---- |
| 2  | Ali  |
| 3  | Sara |

#### UNION

Combines results of two queries and removes duplicates.

```sql
SELECT Name FROM STUDENT_CS
UNION
SELECT Name FROM STUDENT_EE;
```
result:

| Name  |
| ----- |
| Tahir |
| Ali   |
| Sara  |

removes duplicates

#### UNION ALL

Combines results but keeps duplicates.

```sql
SELECT Name FROM STUDENT_CS
UNION ALL
SELECT Name FROM STUDENT_EE;
```
result:

| Name  |
| ----- |
| Tahir |
| Ali   |
| Ali   |
| Sara  |

#### INTERSECT

Returns common rows from both queries.

```sql
SELECT Name FROM STUDENT_CS
INTERSECT
SELECT Name FROM STUDENT_EE;
```

result: 

| Name |
| ---- |
| Ali  |

#### EXCEPT (or MINUS)

Returns rows from the first query not present in the second query.

```sql
SELECT Name FROM STUDENT_CS
EXCEPT
SELECT Name FROM STUDENT_EE;
```

result:

| Name  |
| ----- |
| Tahir |

#### summary

| Operation      | Meaning                         |
| -------------- | ------------------------------- |
| UNION          | Combine and remove duplicates   |
| UNION ALL      | Combine including duplicates    |
| INTERSECT      | Common records                  |
| EXCEPT / MINUS | Records in first but not second |

## HAVING

The HAVING clause is used to filter groups of rows after aggregation (after GROUP BY).

| Dept | Salary |
| ---- | ------ |
| HR   | 40000  |
| HR   | 50000  |
| IT   | 60000  |
| IT   | 70000  |


```sql
SELECT Dept, AVG(Salary)
FROM Employee
GROUP BY Dept
HAVING AVG(Salary) > 50000;
```
result:

| Dept | AVG(Salary) |
| ---- | ----------- |
| IT   | 65000       |

## Replicate table into another

#### Copy Structure + Data

```sql
CREATE TABLE Employee_Copy AS
SELECT * FROM Employee;
```
#### Copy Only Structure (No Data)

```sql
CREATE TABLE Employee_Copy AS
SELECT * FROM Employee WHERE 1=0;
```
- here we can write anything that returns false: for eg 1=0, 2=1, 5<1, etc

#### Copy Data into Existing Table

```sql
INSERT INTO Employee_Copy
SELECT * FROM Employee;
```
#### Copy Specific Columns

```sql
CREATE TABLE Employee_Names AS
SELECT Name FROM Employee;
```
#### Copy Only Some Rows

```sql
CREATE TABLE CS_Employees AS
SELECT * FROM Employee
WHERE Dept = 'CS';
```

## VIEW

A View is a virtual table created from a SQL query.

It does not store data itself; instead, it displays data from one or more tables whenever the view is queried.

```sql
CREATE VIEW frontview AS 
SELECT * FROM student 
WHERE s_branch = 'ece';
```
```sql
SELECT * FROM frontview;
```

## SCALAR function

A Scalar Function is a function that:
- Takes one or more input values
- Returns a single value

e.g 

```sql
SELECT UPPER(Name) FROM Student;
```

UPPER() → scalar function

UPPER can be also written as ucase

takes name: tahir as input

o/p: TAHIR

#### Types of Scalar Functions

1. String Functions

Used for text manipulation.

```sql
SELECT UPPER(Name) FROM Student;
SELECT LOWER(Name) FROM Student;
SELECT LENGTH(Name) FROM Student;
SELECT CONCAT('Mr ', Name) FROM Student;
``` 

concat -> mr + tahir -> mr tahir

2. Numeric Functions

Used for mathematical operations.

```sql
SELECT ABS(-10);
SELECT ROUND(3.567,2);
SELECT CEIL(3.2);
SELECT FLOOR(3.9);
SELECT RAND(); // random number
```

3. Date Functions

Used for date calculations.

```sql
SELECT NOW();
SELECT CURDATE();
SELECT YEAR(NOW());
```

#### 📊 Scalar vs Aggregate Functions

| Feature  | Scalar Function | Aggregate Function    |
| -------- | --------------- | --------------------- |
| Works on | Single value    | Multiple rows         |
| Returns  | Single value    | Single summary value  |
| Example  | UPPER(), ABS()  | SUM(), AVG(), COUNT() |

## adding foreign-key using alter keyword

suppose we have 2 tables , department and student

Department: primary key = DeptID

| DeptID | DeptName |
| ------ | -------- |
| 101    | CSE      |
| 102    | ECE      |

student: 

| ID | Name  | DeptID |
| -- | ----- | ------ |
| 1  | Tahir | 101    |
| 2  | Ali   | 102    |

Here DeptID should reference the DEPARTMENT table.

```sql
ALTER TABLE student
ADD CONSTRAINT fk_dept
FOREIGN KEY (DeptID)
REFERENCES department(DeptID);
```
