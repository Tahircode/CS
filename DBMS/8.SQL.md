## JOIN

A JOIN is used to combine rows from two or more tables based on a related column between them.

Types:
- inner join
- left join
- right join
- full join
- cross join 
- self join

example

student table:

| ID | Name  | DeptID |
| -- | ----- | ------ |
| 1  | Tahir | 101    |
| 2  | Ali   | 102    |
| 3  | Sara  | 101    |

department table

| DeptID | DeptName |
| ------ | -------- |
| 101    | CSE      |
| 102    | ECE      |

We want to show student name with department name.

![alt text](<0.public/image copy 14.png>)

#### INNER JOIN

returns the intersection/matching rows

![alt text](<0.public/image copy 13.png>)

```sql
SELECT student.Name, department.DeptName
FROM student
INNER JOIN department
ON student.DeptID = department.DeptID;
```
| Name  | DeptName |
| ----- | -------- |
| Tahir | CSE      |
| Ali   | ECE      |
| Sara  | CSE      |

#### LEFT JOIN

Returns:
- All rows from left table
- Matching rows from right table
- If no match → NULL

```sql
SELECT student.Name, department.DeptName
FROM student
LEFT JOIN department
ON student.DeptID = department.DeptID;
```

#### RIGHT JOIN

Returns:
- All rows from right table
- Matching rows from left table

```sql
SELECT student.Name, department.DeptName
FROM student
RIGHT JOIN department
ON student.DeptID = department.DeptID;
```

#### FULL JOIN ( FULL OUTER JOIN)
Returns:
- All rows from both tables
- Non-matching rows filled with NULL

```sql
SELECT student.Name, department.DeptName
FROM student
FULL JOIN department
ON student.DeptID = department.DeptID;
```
(MySQL does not support FULL JOIN directly.)

#### CROSS JOIN

Returns cartesian product.

```sql
SELECT *
FROM student
CROSS JOIN department;
```

#### SELF JOIN

Joining a table with itself.

```sql
SELECT A.Name, B.Name
FROM employee A
JOIN employee B
ON A.ManagerID = B.ID;
```

## NESTED QUERY

A Nested Query (or Subquery) is a query inside another SQL query.

The inner query runs first, and its result is used by the outer query

exmple;

Table: student

| s_id | s_name | s_branch |
| ---- | ------ | -------- |
| 1    | Tahir  | CSE      |
| 2    | Ali    | ECE      |
| 3    | Sara   | CSE      |

Suppose we want students from the same branch as Tahir.

```sql
SELECT s_name
FROM student
WHERE s_branch = (
    SELECT s_branch
    FROM student
    WHERE s_name = 'Tahir'
);
```

result:

| s_name |
| ------ |
| Tahir  |
| Sara   |

#### Types of nested query

#### 1. Single Row Subquery

Returns one value.

```sql
SELECT s_name
FROM student
WHERE s_id = (
    SELECT MAX(s_id)
    FROM student
);
```
#### 2. Multiple Row Subquery

Returns multiple values.

Uses operators like:
- IN
- ANY
- ALL

```sql
SELECT s_name
FROM student
WHERE s_branch IN (
    SELECT s_branch
    FROM student
    WHERE s_id > 2
);
```

#### 3. Correlated Subquery ⭐

Inner query depends on outer query.

```sql
SELECT s_name
FROM student s1
WHERE s_id > (
    SELECT AVG(s_id)
    FROM student s2
    WHERE s1.s_branch = s2.s_branch
);
```

#### NOTE:

Nested queries can appear in:
- SELECT
- WHERE
- FROM
- HAVING

Example:

```sql
SELECT *
FROM student
WHERE s_id > (
    SELECT AVG(s_id)
    FROM student
);
```

## DUAL TABLE

The DUAL table is a special dummy table used in SQL when you want to execute a query without referring to any real table.

It contains only one row and one column.

examples:

```sql
SELECT 5 + 3 FROM DUAL;
```

```sql
SELECT NOW() FROM DUAL;
```

```sql
SELECT UPPER('tahir') FROM DUAL;
```

DUAL is used when:
- Testing expressions
- Running functions
- Performing calculations
- No actual table data is required

In MySQL, the DUAL table is optional.

You can write:

```sql
SELECT 5+3;
```
instead of

```sql
SELECT 5+3 FROM DUAL;
```
But in some databases like Oracle, FROM DUAL is required.

#### 2. GROUP BY VS DISTINCT

```sql
select name from student group by name;

select distinct(name) from student;
```

both will give me the same result

DISTINCT

DISTINCT is used to remove duplicate rows from the result.

👉 It returns unique values only.

GROUP BY

GROUP BY is used to group rows with the same values so that aggregate functions can be applied.

#### 7. (Important): find Nth heighest salary

```sql
SELECT salary
FROM employee
ORDER BY salary DESC
LIMIT row_skip, row_count;
```

DESC -> heightest to lowest

row_skip -> rows to be skipped

row_count -> returns the row_count number of rows

e.g;

```sql
SELECT salary
FROM employee
ORDER BY salary DESC
LIMIT 2, 1;
```

for Nth row

```sql
SELECT salary
FROM employee
ORDER BY salary DESC
LIMIT N-1, 1;
```
